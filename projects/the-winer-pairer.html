The Wine Pairer

In our household, wine is a popular topic. We love drinking it (naturally) but also learning about it. My learning it's purely self-indulgent: “how can we enjoy this wine the most” rather than anything more philosophical. With this, two things come front and centre:

  1. Setting - occasion, company, location
  2. Food

Knowing what food to pair with a wine is a skill, and remembering your favourite combinations is often just as tricky. I therefore wanted to build a simple app to save my favourite wines and also play around with some JavaScript. Enter The Wine Pairer!

The process for creating the app was quite simple:

  1. See if an API exists

    From my research I discovered that it's rather slim pickings on food and wine pairing APIs. Perhaps not surprising. However, I eventually found one from Spoonacular to use as my base. The app pulls pairing suggestions from Spoonacular and allows you to save them to your favourites. You can also add to the list manually with a simple form.

  2. Mock up a design

    Thanks to Figma and the fact I wanted this to be a single page, it was a nice quick job. I wanted something a little retro as you can see from the design and product.

  3. Create an MVP, then expand

    First and foremost this was surfacing the right data, utilising fetch, and saving this locally. I had a very rudimentary app that I then worked to flesh out, handling errors and edge cases.

  4. Style

    Once the functionality was doing what I had hoped, I styled the app, keeping to my Figma design in large. I amended certain elements once I saw them in real-life to provide a better UX.

  5. Test & improve

    After testing myself I simply handed it over to my husband and asked him to try to break it. Thankfully, he couldn't. We then ran over what could be better. I did the same exercise with my tech mentor. The consensus was:

    • Duplicate handling - it absolutely didn't make sense to let the user put the same pairings in the same list multiple times
    • Letting the user know if the pairing had been added to the list or, if not, why not
    • Expanding the wine list - this was a minor point but we wanted to expand to include some more niche wines. In reality there are too many wines for a simple drop down so there is more work to do here

  6. Refactor

    While I made an effort to keep my code DRY, there were repetitive elements so I spent a couple of hours tidying up.

Next steps

At present I've kept the wine list very short in the drop down, although you can add any manner of favourites. While it's very easy to expand this, the API does not have extensive pairing. I’d ultimately like to find (or build) another API - Spoonacular has been great for the MVP but I would like something more comprehensive, and which accounts for non-US linguistics, in the future.
